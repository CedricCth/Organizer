# Diagrams

Rendered architecture diagrams for the Couples App. The PNGs in this folder are the canonical rendered versions referenced from [`../docs/ARCHITECTURE.md`](../docs/ARCHITECTURE.md).

| File | What it shows |
|---|---|
| `1_context.png`     | C4 Level 1 — System Context (people + system + external services) |
| `2_container.png`   | C4 Level 2 — Containers (Next.js PWA, Vercel, Supabase) |
| `3_component.png`   | C4 Level 3 — Components inside the Next.js app, with the dependency-rule layers |
| `4_seq_signup.png`  | Sequence — Signup |
| `5_seq_pairing.png` | Sequence — Couple pairing (invite + redeem) |
| `6_seq_todo.png`    | Sequence — Create todo with realtime fan-out |

## Re-rendering

The diagrams are generated by a single Python script — `render.py` — using only `matplotlib` and the standard library. There is no external server, browser, or proprietary tool involved; everything is plain text in the repo.

```bash
cd diagrams
python3 render.py
```

That writes (and overwrites) all six PNGs in this directory.

## Editing

Each diagram is a `render_*` function in `render.py`. The shared style is in the constants and helper functions at the top of the file. To change a diagram, edit the relevant function and re-run.

## Why not Mermaid / draw.io / PlantUML?

The original [`../docs/ARCHITECTURE.md`](../docs/ARCHITECTURE.md) uses Mermaid for in-document rendering on platforms that support it (GitHub, etc.). The PNGs in this folder exist for environments that don't render Mermaid, and for sharing the architecture as an image (e.g., dropping into a slide).

`matplotlib` is the renderer because:
- It runs offline with no Chromium / Java / network dependencies
- It gives precise control over the visual style (warm palette, rounded boxes, etc.)
- It produces deterministic output — re-running the script yields byte-identical PNGs given the same inputs
